









Title: User Registration and Login API

As a registered user
I want to create an account and log in using my email and password
So that I can securely access the application

Functional Requirements

The system shall provide a REST API built using ASP.NET Core.

The system shall allow a user to register with the following details:

First Name (required, max 50 characters)

Last Name (required, max 50 characters)

Email (required, unique, valid email format)

Password (required, minimum 8 characters, must be hashed)

The system shall allow a user to log in using email and password.

The system shall return a JWT token upon successful login.

The system shall prevent registration with an already existing email.

The system shall return appropriate HTTP status codes and error messages.

API Endpoints

POST /api/auth/register

POST /api/auth/login

Validation & Error Handling

Return 400 Bad Request for validation errors.

Return 401 Unauthorized for invalid login credentials.

Return 201 Created when registration is successful.

Return 200 OK when login is successful.

Non-Functional Requirements

Use ASP.NET Core Web API.

Follow clean architecture / layered architecture:

Controller

Service

Repository

Model / DTO

Use Entity Framework Core with an in-memory or SQL Server database.

Use dependency injection.

Passwords must be hashed and salted.

Use DTOs for request and response models.

Acceptance Criteria

A user can successfully register with valid details.

A user cannot register with an existing email.

A user can log in with correct credentials.

A user cannot log in with incorrect credentials.

Passwords are never stored in plain text.

API responses follow REST conventions.

Out of Scope (Optional)

Email verification

Password reset

Role-based authorization

UI / Frontend
